  Lionhead's genius was never the AI. It was the RECEIPTS.

  In Fable, if you kicked a chicken in Oakvale, a farmer mentioned
  it three hours later. The player didn't think anyone was watching.
  That's the magic â€” the feeling of being watched by a world that
  cares about what you did.

  Our version: every choice the player makes â€” even ones that seem
  cosmetic or throwaway â€” gets logged to a MEMORY LEDGER. We don't
  need to USE every entry right away. We just need to HAVE it, so
  that when we write dialogue or trigger events, we can reach back
  and pull something out that makes the player say "holy shit, it
  remembered that."

  The key insight: TRACK EVERYTHING. USE IT LATER.
  Don't design the payoff first. Design the tracking first.
  The payoffs will suggest themselves.

  Every NPC dialogue line has two parts:
    1. BASE LINE â€” the generic version
    2. MEMORY OVERRIDE â€” a version that references history

  The dialogue system checks the Memory Ledger FIRST.
  If it finds a relevant, unsurfaced memory, it uses the override.
  Then it marks that memory as surfaced: true.

  STRUCTURE:

  function getDialogue(character, context, ledger) {
    // 1. Check for unsurfaced memories relevant to this context
    const memories = ledger.filter(m =>
      !m.surfaced &&
      m.actor === relevantActor &&
      MEMORY_TRIGGERS[context]?.includes(m.tag)
    );

    // 2. If memory found, use the memory-aware line
    if (memories.length > 0) {
      const memory = memories[0];
      memory.surfaced = true;
      return MEMORY_LINES[character][memory.tag](memory);
    }

    // 3. Otherwise, use the generic line
    return pick(BASE_LINES[character][context]);
  }
